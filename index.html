Key Features

Standardized API

Trellis defines a common specification for how conformant platforms provide and consume data, tailored to fresh produce & supply chains.

Privacy & Permissions

Sensitive information is only shared with authorized partners. You control who sees what, every step of the way.

Not Centralized

Data flows directly between participants— no central server storing all your data. Keep ownership firmly in your control.

Automatic Synchronization

Whenever produce data changes in your Trellis-conformant system, authorized partners see updates instantly.

How It Works

  1. Adopt the Trellis API: Implement our specification to provide & request data in a uniform format. (Reference implementation: OADA Server)
  2. Permissions First: Request and grant data access through Trellis’s robust permissioning model—companies see only what you allow.
  3. Run Anywhere: Use Docker or Kubernetes to run your own Trellis instance (OADA-based) on-premises or in the cloud.
  4. Real-Time Sync: Changes in your system are instantly reflected in the systems of your authorized network, creating a live produce data ecosystem.
